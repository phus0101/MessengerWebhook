## Root Cause Analysis

### Primary Issue: State Transition Logic Broken

**Evidence from logs:**
```
[21:06:14 INF] Product Kem Chong Nang (Code: KCN) found directly from message
[21:06:14 INF] Setting selectedProductCodes to [KCN]
[21:06:14 INF] Saved 1 entities to in-memory store.
[21:06:14 INF] Saved 1 entities to in-memory store.
```

**Analysis:**
- Product selection works correctly
- Data persistence works
- State transition from `Idle` → `Consulting` happens
- **BLOCKER:** Transition from `Consulting` → `CollectingInfo` never occurs

**Hypothesis Chain:**

1. ❌ **Phase 0 changes broke state logic**
   - Eliminated: Phase 0 only touched greeting strings and personality traits
   - No state machine logic modified
   - Changes in `CustomerIntelligenceService.cs` (lines 99-103) only affect greeting instruction codes
   - Changes in `SalesStateHandlerBase.cs` (lines 581-649) only renamed method and added `VipTier.Returning` handling

2. ✅ **Pre-existing bug in state transition conditions**
   - Confirmed: State machine not advancing after product selection
   - Likely cause: Missing or incorrect condition check in `SalesStateHandlerBase`
   - Need to examine state transition logic in `HandleConsultingAsync` or similar method

### Secondary Issue: Cache Configuration Error

**Error:**
```
System.InvalidOperationException: Cache entry must specify a value for Size when SizeLimit is set.
   at Microsoft.Extensions.Caching.Memory.MemoryCache.SetEntry(CacheEntry entry)
   at MessengerWebhook.Services.DraftOrderCoordinator.FinalizeDraftOrderAsync
```

**Location:** `DraftOrderCoordinator.cs:76`

**Analysis:**
- Memory cache configured with `SizeLimit` but entries don't specify `Size`
- Prevents draft order creation
- Causes order confirmation to fail

### Tertiary Issue: Instruction Code Leaking to Output

**Evidence:**
```
Actual reply: "VIP_WARM_GREETING\n\nEm len thong tin cho Kem Lua roi nha..."
```

**Analysis:**
- Phase 0 changed hardcoded greetings to instruction codes
- Instruction codes appearing in actual bot responses
- **Root cause:** `GeminiService` not replacing `{PERSONALITY_TRAITS}` placeholder or instruction codes not being processed
- Need to verify prompt template replacement logic

### Warning: AI Extraction Failures

**Repeated warning:**
```
[21:06:14 WRN] AI extraction failed, falling back to regex
System.Text.Json.JsonException: 'D' is an invalid start of a value.
```

**Analysis:**
- AI-based contact extraction consistently failing
- Regex fallback working correctly
- Non-critical but indicates Gemini response format issue
- May need prompt adjustment or response parsing fix
